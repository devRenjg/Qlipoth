# ui-theme Specification

## Purpose
平台统一视觉风格——深空蓝科技主题，营造"大型活动保障作战"的专业严谨感，同时保障可读性，不改动既有功能色与布局。

## Requirements

### Requirement: 深空蓝导航与全局点缀

平台 SHALL 采用统一的深空蓝视觉主题：顶部导航深蓝渐变底+白字+选中态亮蓝高亮；内容区淡冷灰蓝背景，卡片淡蓝灰边框+蓝调阴影+hover 泛蓝光，页面主标题左侧蓝青渐变竖条。主题点缀 SHALL 只作用于中性外观，不改变布局、不改变功能色（标签/状态/按钮）、不降低正文可读性（正文保持深色、背景仅极淡冷灰）。

#### Scenario: 统一风格且可读

- **WHEN** 用户浏览任意内容页
- **THEN** 看到一致的深空蓝卡片风格与导航高亮，正文清晰可读，原有标签/状态色不受影响

### Requirement: 卡片折叠与红色告警语义保留

作战地图与保障清单的分组卡片 SHALL 默认折叠、点击展开并带顺滑动效，采用统一蓝灰圆角卡片风格。保障清单 SHALL 严格保留红色告警语义：事故/故障维度红色标题与左红条、严重度 P0 红 / P1 橙 / P2 灰、"当天发生·务必重视"深红标签——不被主题色覆盖。

#### Scenario: 告警色不被主题覆盖

- **WHEN** 保障清单套用深空蓝卡片风格
- **THEN** 事故/故障维度、P0、当天发生等仍以红色醒目呈现，告警语义不丢失
